Security
Procedures

Security procedures ensure consistency in the implementation of a security control or execution of a security. Find out what are these procedures to secure your devices.

by Alexander Oxales

Security Policy

Security policy is a set of security objectives that ensure the security of a network, the data
and the computers in an organization.

These include policies for Identification and Authentication, Passwords,
Acceptable Use, Remote Access, Network Maintenance, and Incident Handling.

Why do we need to secure our devices?

Ensure a safe network
environment

Protect assets

Prevent theft, vandalism,
and data loss

Physical Security

Physical security is also important as data can be stolen with the computer.

These include policies for Access to an organization's premise
Access to restricted areas and Computing and network infrastructure.

Types of Secure Locks

Conventional lock - Requires a key into the door handle mechanism.
Deadbolt lock - Requires entering the key into a lock separate
from the door handle mechanism.
Electronic lock- Requires entering a PIN combination into the keypad.
Token-based lock - Requires a secure card by tapping or swiping.
biometric lock- Requires reading fingerprint by using a biometric scanner.
Multifactor lock - A lock that uses a combination of the above mechanisms.

How to Secure Data

Data is likely to be an organization's most valuable assets.
It can be lost or damaged in circumstances such as theft, equipment failure, or a disaster. We can protect our data using data backups, file/folder encryption and permissions.

DATA BACKUP

Backing up data is one of the
most effective ways of protecting against data loss

A data backup stores a copy of the information
on a computer to removable backup media and should be performed on a daily basis.

Windows hosts have a backup and restore utility while macOS devices have Time Machine Utility for backup.

FILE AND FOLDER PERMISSIONS

Permissions are rules you configure to limit access for users.

Users should be limited to only the resources they need in a computer or on a network.

There are basically six types of permissions in Windows:
Full Control, Modify, Read & Execute, List Folder Contents, Read, and Write.

FILE AND FOLDER ENCRYPTION

File encryption protects individual files
or file systems by encrypting them with a specific key, making them accessible only to the keyholder.

The goal is to prevent malicious or unauthorized parties from accessing files that are stored on the disk.

Encrypting File System (EFS) is a Windows feature
that can encrypt data.

DATA WIPING

Protecting data also includes removing files from storage devices when they are no longer needed.

Software tools can be used to recover folders, files, and even entire partitions.

Other storage media and documents can be destroyed by shredding or incineration.

HARD DRIVE
RECYCLING AND DESTRUCTION

When a storage media is no longer needed,
the media can either be:

Destroyed - Destroying the hard drive fully ensures that data cannot be recovered from a hard drive.

Recycled - Hard drives that have been wiped can be reused in other computers. The drive can be reformatted, and a new operating system installed.
